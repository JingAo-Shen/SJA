前后端分离 教学讲义

Spring Boot + 前端（React/Vue） | 单体模块化 | 学生自学版
技术栈 阶段化教学

技术栈

后端：Java 17、Spring Boot、Spring Data JPA。前端：React（Vite）或 Vue 3（Vite）。数据库：Postgres/MySQL（课堂可用 H2）。容器：Docker + Docker Compose。测试：JUnit5、Mockito。

阶段化教学（每步均需完整交付）

1. 需求（完整验收用例）

目标：输出完整验收用例，覆盖正常、边界、异常与性能场景。
【Prompt】 把需求写清，输出： 1) 一句话功能概述 2) 完整验收用例文档（分组、每条含输入/预期/步骤/优先级） 3) MVP 功能清单 4) 风险与假设说明
输出：需求文档、完整验收用例、MVP、风险说明

2. 接口（完整 RESTful 列表）

目标：列出所有 REST 接口及错误矩阵与鉴权与性能备注。
【Prompt】 列出完整 RESTful 接口清单：方法、路径、参数、Body、成功/失败响应、错误码说明、鉴权要求、性能备注。 输出：RESTful-APIs.md、示例请求/响应 JSON、curl 示例。
输出：完整接口文档、示例 JSON、curl 示例

3. 原型（高保真设计说明）

目标：输出高保真设计说明，包含页面、组件、交互、视觉规范与示例数据。
【Prompt】 根据需求与接口，输出完整高保真设计说明：页面清单、组件与交互、视觉规范、示例数据、导出资源清单、无障碍与移动适配说明。 输出：high-fidelity-design.md + 每页示例数据 JSON。
输出：高保真设计说明、示例数据、资源清单

4. 后端（完整实现）

目标：实现接口文档中全部路由，包含校验、异常、权限与性能注释。
【Prompt】 根据接口文档，生成完整 Spring Boot 项目代码：Entity/Repo/Service/Controller、全局异常、校验、application.yml、pom.xml、性能说明。 输出：后端代码文件夹、配置与运行说明。
输出：后端代码库、配置、运行说明

5. 前端（完整对接）

目标：实现所有页面并精确对接后端，包含鉴权、加载/错误/空态处理。
【Prompt】 实现前端项目（React/Vue）：组件化、API 封装、鉴权、响应式与无障碍支持。输出：前端代码库、构建与运行说明。
输出：前端代码库、运行说明、对接记录

6. 测试（覆盖 + 报告）

目标：实现单元、集成与性能测试并输出报告与覆盖率。
【Prompt】 生成完整测试套件：单元测试、集成测试（H2/Testcontainers）、性能测试脚本。输出：测试代码、脚本、报告、覆盖率结果。
输出：测试代码、报告、性能脚本

7. 打包与部署（Docker + CI）

目标：生成 Docker 与 CI，使合并后自动构建与测试。
【Prompt】 生成发布方案：后端/前端 Dockerfile、docker-compose.yml、.github/workflows/ci.yml、部署与回滚说明。输出：发布文件夹与说明文档。
输出：Dockerfile、docker-compose、CI 配置、部署说明
本文档为学生自学最终版，按阶段逐项交付完整产物。
